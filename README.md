# 🦙 Llama Launcher — Простой Запускатор ИИ

*[Russian-language GUI launcher for llama-server (llama.cpp). See below for details.
No installation needed for end users — portable .exe, no console required.]*

Простое окошко с кнопками и ползунками для запуска [llama-server](https://github.com/ggml-org/llama.cpp)
без консоли и без ручного набора команд. Сделано специально для тех, кто не хочет
(или не умеет) разбираться с параметрами командной строки — включая детей и
абсолютных новичков.

## Возможности

- Выбор `llama-server.exe` и файла модели `.gguf` через диалоговое окно
- Пресеты поведения модели одним кликом: Точный / Обычный / Творческий
- Ползунки для потоков CPU, размера контекста, GPU-офлоада, batch/ubatch
- Всплывающие подсказки над каждым параметром — наведи мышкой и узнай, за что он отвечает
- Автоопределение количества ядер CPU и объёма RAM — разумные значения по умолчанию сразу при первом запуске
- Выбор браузера, в котором открывать чат (Chrome / Edge / Firefox / свой вариант)
- Большие кнопки "Запустить" и "Стоп", лог сервера прямо в окне
- Настройки сохраняются между запусками

## Как собрать portable .exe

Собирается на Windows в один шаг:

```bash
pip install pyinstaller
pyinstaller --onefile --windowed --name "LlamaLauncher" LlamaLauncher.py
```

Готовый файл появится в `dist/LlamaLauncher.exe` — его можно переносить
на любой компьютер, установка Python на целевой машине не требуется.

## Как пользоваться

1. Запусти `LlamaLauncher.exe`
2. Кнопкой "Обзор..." укажи `llama-server.exe` и файл модели `.gguf`
3. Выбери пресет ответа (Точный / Обычный / Творческий)
4. Нажми зелёную кнопку "ЗАПУСТИТЬ"
5. Через пару секунд само откроется браузер с чатом

Заметка про параметры (кратко)
--------------------------------
- Потоки (-t): по числу физических ядер процессора.
- Контекст (-c): "память" диалога в токенах, больше = больше RAM.
- Слоёв на GPU (-ngl): 0, если видеокарта слабая/старая или её нет.
- --mlock: не даёт системе выгружать модель из RAM на диск.
- Flash Attention (-fa): ускоряет и экономит память, стоит держать
  включённым почти всегда.


## Требования

- Python 3.8+ (только для сборки; готовый `.exe` не требует Python)
- Только стандартная библиотека Python — никаких сторонних зависимостей для самого GUI
- Windows (кроссплатформенность для Linux/macOS не тестировалась, в теории должен запускаться и там как обычный `.py`-скрипт)
